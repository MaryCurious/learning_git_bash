Структура баг-репорта: 

Summary: Заголовок
Project: Название проекта
Enviroment: Окружение, на котором обнаружен баг
Component: Компонент системы, где обнаружен баг
Severity: Влияние бага на работоспособность приложения
Priority: Степень важности найденного бага и очередность его выполнения с точки зрения бизнеса
Status: Статус бага в жизненном цикле бага
Description: Описание бага, шаги, ведущие к нему
Expected: Result Ожидаемый результат
Actual: Result Фактический результат
Author: Тестировщик, обнаруживший баг 
Attachment: Сриншоты/видео, другие вспомогательные файлы
